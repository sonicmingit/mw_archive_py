## 1. 用户需求记录（原始需求 + 初步思路）
- 需求描述：
  - 降低单 cookie 被限流或失效导致的整体中断。
- 用户初步实现思路：
  - cookie 池状态机：`active` / `cooldown` / `invalid`
  - 调度策略：轮询 + 失败退避
  - 健康检查：周期验证可用性并统计成功率
- 目标效果：
  - 单 cookie 异常不阻塞整体归档任务
- 约束/注意事项：
  - 失败判定标准（限流/验证/网络）要可配置
  - 状态切换需有日志，便于排查

## 2. AI 分析（难度 / 思路 / 改动范围）
- 实现难度：中
- 总体实现思路：
  - 引入 cookie 池管理器，封装状态与选择算法
  - 在归档请求入口统一通过池子获取可用 cookie
  - 增加健康检查任务和后台统计接口
- 预计改动模块：
  - `app/server.py` 归档入口
  - cookie 配置与持久化结构
  - 配置页和监控展示

  - 状态误判导致健康 cookie 被错误降级

- 验证方案：
  - 注入失效 cookie、限流 cookie、正常 cookie 进行混合测试
  - 验证自动切换和恢复流程
