## 1. 用户需求记录（原始需求 + 初步思路）
- 需求描述：
  - 降低单 cookie 被限流或失效导致的整体中断。
- 用户初步实现思路：
  - cookie 池状态机：`active` / `cooldown` / `invalid`
  - 调度策略：轮询 + 失败退避
  - 健康检查：周期验证可用性并统计成功率
- 目标效果：
  - 单 cookie 异常不阻塞整体归档任务
- 约束/注意事项：
  - 失败判定标准（限流/验证/网络）要可配置
  - 状态切换需有日志，便于排查
- 失效cookie标记
  - 如果某个cookie失效,在页面进行标记(可以修改cookie.json文件格式进行记录)
- 增加国内/国际cookie两组记录
  - 增加对国际服的支持,单独设置cookie,归档根据域名正则判断使用对应cookie
      - 国内: https://makerworld.com.cn/zh/models/*
      - 国际: https://makerworld.com/zh/models/*
- 接口兼容:
  - 旧的设置cookie接口保留,默认之前格式设置第一个国内cookie
  - 创建新接口,支持设置国际和国内多cookie.(这个更新文档是只写支持国内国际,不写支持多个)
- 页面修改:
  - 分开显示国内/国际cookie设置
  - 默认各显示一个配置,加一个添加按钮可设置多个
- 隐藏功能
  - 在app\version.yml配置文件中添加一个参数,控制开启关闭多cookie功能(保留国内国际逻辑,但只使用第一个cookie),默认关闭,开启后页面开放设置多cookie. 其实就相当于控制页面添加cookie开关按钮
  - 代码可以不修改,相当于用户只设置了一个cookie

## 2. AI 分析（难度 / 思路 / 改动范围）
- 实现难度：中
- 总体实现思路：
  - 后端将 Cookie 存储升级为结构化 `cookie.json`，按 `cn/global` 两个平台分组。
  - 以版本开关 `app/version.yml -> multi_cookie_enabled` 控制“多 Cookie”是否对用户开放。
  - 在开关关闭时，代码仍保留国内/国际分组逻辑，但每个平台只使用第一个 Cookie，保持旧使用习惯。
  - 在开关开启时，归档和重下载链路按平台执行 Cookie 轮询，并在出现 401/403/Cloudflare/429 等典型场景时更新状态。
  - 配置页拆分为“国内平台 Cookie / 国际平台 Cookie”两组输入，并在页面显示当前状态标记。
- 预计改动模块：
  - `app/server.py` 归档入口
  - Cookie 配置与持久化结构
  - 配置页 Cookie 管理区域
  - `app/version.yml` 版本功能开关
- 主要风险点：
  - 如果错误分类过宽，会把非 Cookie 错误误判为失效或冷却。
  - 缺失 3MF 重试、实例重下、模型重下如果不跟随平台切换，仍会退回单 Cookie 行为。
  - 配置页改造必须兼容旧接口，避免影响现有用户直接保存 Cookie。
- 验证方案：
  - 检查 `cookie.json` 自动迁移/自动生成逻辑是否正常。
  - 检查旧接口 `POST /api/cookie` 是否仍默认写入国内平台第一个 Cookie。
  - 检查 `multi_cookie_enabled=false` 时页面是否保留国内/国际双分组，但不暴露“添加”按钮。
  - 检查归档与重下载链路是否统一走平台 Cookie 选择逻辑。

- 关键实现点：

    - `cn[] / global[]`
    - 每项包含 `value / status / success_count / failure_count / cooldown_until / last_error` 等字段
  - 新增平台识别逻辑：
    - `makerworld.com.cn` -> `cn`
    - `makerworld.com` -> `global`
  - 新增 Cookie 轮询/失败切换逻辑：
    - 开关关闭：仅使用每个平台第一个 Cookie
    - 开关开启：按轮询顺序尝试可用 Cookie

    - 429 -> `cooldown`
    - 冷却到期后自动恢复 `active`
  - 当前配置页始终保留国内 / 国际两组 Cookie 配置：
    - `multi_cookie_enabled=false` 时，不开放“添加 Cookie”，每组只使用第一个 Cookie
    - `multi_cookie_enabled=true` 时，开放同平台多条 Cookie 输入和轮询

  - 旧接口 `POST /api/cookie` 保留，默认仍写入国内平台第一个 Cookie
  - 新增 `GET/POST /api/cookies`，用于国内/国际 Cookie 组管理
  - 配置页已拆成国内/国际两组 Cookie 区域，并显示状态标记
  - 单平台单 Cookie 模式下标题显示为 `Cookie`，不显示 `Cookie 1`

    - 站点主页返回 `403` 会造成误判
    - 状态仅由真实归档 / 重下载链路驱动更新
