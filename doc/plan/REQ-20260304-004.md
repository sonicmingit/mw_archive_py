## 1. 用户需求记录（原始需求 + 初步思路）
- 需求描述：
  - 在在线模型页面详情直接编辑归档内容并保存。
- 用户初步实现思路：
  - 可编辑字段：标题、标签、说明、分类、版本备注
  - 可以修改对应图片,增加,替换,删除. 可选择或单独设置封面图片
  - 字段校验：长度、必填、非法字符
- 目标效果：
  - 在线 UI 可直接维护已归档内容，保存后展示与检索即时一致
- 约束/注意事项：
  - 保证写入安全，避免 `meta.json` 被破坏 (可以做一份备份)
  - 回滚机制需简单可靠

## 2. AI 分析（难度 / 思路 / 改动范围）
- 实现难度：中
- 总体实现思路：
  - 新增编辑 API（读取、校验、更新）
  - 前端增加编辑入口与保存交互
  - 增加简单版本快照（例如 `meta.history/` 或日志）
- 预计改动模块：
  - `app/server.py` 模型编辑相关 API
  - `app/static/js/model.js` 或配置页编辑组件
  - 数据落盘和版本记录策略
- 主要风险点：
  - 并发写入冲突
  - 回滚范围定义不清导致恢复异常
- 验证方案：
  - 编辑多字段后刷新校验
  - 触发非法输入验证
  - 执行一次回滚并核对内容

## 3. AI 实现记录（实际落地）
- 实际改动文件：
  - `app/server.py`
  - `app/templates/model.html`
  - `app/static/js/model.js`
  - `app/static/css/model.css`
- 关键实现点：
  - 新增模型在线编辑 API，支持标题、标签、说明、分类、版本备注保存
  - 新增设计图管理能力，支持保留/删除已有图片、追加新图片、重新指定封面
  - 新增 `meta.json` 自动备份与“恢复最近备份”能力
  - 详情页右侧信息卡新增编辑入口，编辑弹窗支持富文本简介编辑
  - 最近备份时间前端格式化为可读日期时间
- 与预期差异：
  - 图片“替换”采用“删除旧图 + 上传新图”的交互实现，未单独做逐张替换接口
- 测试/验证结果：
  - 使用 Playwright MCP 打开主页、配置页、在线详情页，主要内容显示正常
  - 实测在线编辑保存成功，详情页字段刷新正常
  - 实测恢复最近备份成功，修改内容可回滚
- 后续待办：
  - 可继续优化简介编辑体验，例如补充图片插入、源码切换或更完整的富文本工具栏
  - 可进一步消除详情页 `screenshot.png` 缺失时的 404 噪音
