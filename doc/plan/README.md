# 需求开发记录说明与模板

本目录用于按“**一个需求一个文档**”记录从提出到实现的全过程，便于后续回顾与继续开发。

## 目录规则

- 文件命名：`REQ-YYYYMMDD-序号.md`
- 每个文件只记录一个需求
- 需求文档不写里程碑拆分，聚焦“需求内容 / AI分析 / AI实现记录”
- 汇总 `README.md` 底部索引需为每条记录增加简单状态标记：`✅` 表示已完成，`🟡` 表示未完成

## 需求状态说明

- `🟡 待分析`：已提出需求，等待 AI 分析
- `🛠️ 开发中`：已进入实现
- `🧪 待验证`：已实现，等待回归或用户验证
- `✅ 已完成`：验证通过并完成记录
- `⚪ 已关闭`：确认不做或合并到其他需求

```markdown

- 提出时间：YYYY-MM-DD HH:mm
- 完成时间：YYYY-MM-DD HH:mm（未完成填：-）
- 提出人：姓名或ID
- 最后更新：YYYY-MM-DD
- 关联版本：vX.Y.Z（未发布填：待发布）
```
