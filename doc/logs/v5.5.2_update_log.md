# v5.5.2 更新日志

> 更新日期: 2026-03-13

## 更新概述
本次版本聚焦“本地 3MF 文件预整理”和“模型页归档插件增强”。配置页新增独立的本地整理入口，可以对固定目录中的 `3MF` 文件执行一键整理，按模型归类、按配置重命名、分离重复文件，并输出整理耗时与文本报告。同时补齐了 Docker 挂载目录、插件国际站支持与使用说明，便于在本地和容器环境中统一使用。

## 当前版本同步说明
- 当前版本：`v5.5.2`
- 更新说明：[`doc/logs/v5.5.2_update_log.md`](./v5.5.2_update_log.md)
- 本次重点：
  - 配置页新增本地 3MF 整理入口；
  - 支持固定目录一键整理、重复文件分离与耗时统计；
  - Docker 新增独立整理目录挂载；
  - Chrome 插件与油猴脚本支持国际站模型页归档，扩展入口同步精简并增加明显归档提示。

## 详细变更

### 新增功能
- 新增本地 `3MF` 整理能力：
  - 支持扫描固定目录中的全部 `3MF`
  - 支持按模型归类到独立目录
  - 支持按配置名重命名文件
  - 支持将重复文件单独放入“重复文件”目录
- 新增配置页整理入口：
  - 支持单独保存整理目录与处理模式
  - 支持在配置页直接点击“开始整理”
  - 支持显示最近整理记录、耗时与报告路径
- 新增插件国际站支持：
  - Chrome 插件支持 `makerworld.com` 国际站模型页注入归档按钮
  - 油猴脚本支持 `makerworld.com` 国际站模型页归档
- 新增公共整理模块：
  - 页面入口与独立脚本共用同一套整理逻辑
  - 降低后续维护时规则分叉风险

### 优化改进
- 整理目录命名规则优化：
  - 有 `DesignModelId` 的模型目录命名为 `MW_<作者名>_<模型名>`
  - 无 `DesignModelId` 的模型目录命名为 `Others_<作者名>_<模型名>`
  - 若缺少作者信息，则不拼接作者段
- 整理结果输出优化：
  - 增加单次整理耗时统计
  - 文本报告中同步展示模型数、配置数、重复数、失败数与耗时
- 状态展示优化：
  - 整理记录按整理根目录分开存储
  - 配置页只显示当前整理目录对应的最近记录和最近报告
- 插件交互优化：
  - Chrome 扩展弹窗移除“同步 Cookie”和“打开本地模型”按钮
  - “本地模型主页”按钮放大并保留为独立入口
  - 扩展弹窗和页面浮动按钮均增加更明显的归档开始 / 完成提示

### Bug 修复
- 修复配置页触发整理时，相对路径未按 `app/` 目录解析的问题
- 修复配置页“最近报告”误显示其他目录历史报告路径的问题
- 修复配置标题中包含小数点时，文件名被错误截断为 `0.3mf` 的问题
- 修复 Chrome 扩展弹窗内使用原生 `alert()` 导致归档时界面卡死的问题

## 兼容性与注意事项
- 新增默认整理目录 `app/organize/`，Docker 部署时建议单独挂载
- 整理功能会自动忽略历史输出目录：
  - `整理完成`
  - `重复文件`
  - `整理失败`
  - `整理报告`
  - 以及旧版 `_organized/_duplicates/_failed/_reports`
- 若是旧状态文件，程序会自动兼容；但配置页只有在当前整理目录实际执行过一次整理后，才会显示该目录自己的最近报告
