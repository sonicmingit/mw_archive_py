# v5.4.0 更新日志

> 更新日期: 2026-03-06

## 更新概述
本次为本地模型管理增强版本，重点新增本地 `3MF` 批量导入、监控目录扫描、导入汇总推送与最近扫描记录展示。批量导入逻辑与在线归档保持隔离，只作用于 `LocalModel_*`，不会与 `MW_*` 在线归档模型混合。

## 当前版本同步说明
- 当前版本：`v5.4.0`
- 更新说明：[`doc/logs/v5.4.0_update_log.md`](./v5.4.0_update_log.md)
- 本次重点：
  - 新增本地 `3MF` 批量导入与监控目录能力；
  - 本地批量导入支持推送汇总、耗时统计、最近扫描记录与最近 5 次成功记录；
  - 监控目录默认 `./watch`，导入后自动归档到 `_imported/` / `_failed/` 并忽略后续扫描。

### v5.4.0 配置与升级说明
如果你是从旧版本升级到 `v5.4.0`，这次重点增加了本地批量导入配置，建议同时检查 `app/config/` 和 `app/watch/`：
- 推荐继续挂载：
  - `./app/data:/app/data`
  - `./app/logs:/app/logs`
  - `./app/config:/app/config`
- 推荐新增挂载：
  - `./app/watch:/app/watch`
- 新增配置能力：
  - `app/config/config.json` 中新增 `local_batch_import`
  - 默认监控目录为 `./watch`
  - 成功 / 重复文件自动移动到 `./watch/_imported/`
  - 失败文件自动移动到 `./watch/_failed/`
- 页面入口：
  - `/config` 页面可配置监控目录、扫描间隔、解析并发、推送开关
  - 手动导入弹窗新增“选择目录批量导入”

### v5.4.0 大批量导入建议
- 小批量文件可以直接使用“选择目录批量导入”。
- 大批量文件建议优先把 `3MF` 先放入 `watch` 目录，再在页面点击“立即扫描并导入”。
- 这样可以避免浏览器批量上传耗时过长，也更适合 Docker 映射目录场景。

## 详细变更

### 新增功能
- 新增本地批量导入服务层：
  - 支持递归扫描监控目录中的 `3MF`
  - 支持识别同模型不同配置并聚合到同一个 `LocalModel_*`
  - 支持手动目录上传批量导入与监控目录批量导入两种入口
- 新增本地批量导入监控能力：
  - 配置页可开启 / 关闭后台 watcher
  - 关闭时不启动后台线程，不额外占用资源
  - 默认监控目录为 `./watch`
- 新增导入状态与页面展示：
  - 最新扫描记录
  - 最近 5 次成功记录
  - 导入耗时、处理数、成功 / 跳过 / 失败统计
- 新增导入完成推送：
  - 区分“监控目录导入完成”和“手动目录导入完成”
  - 推送中增加耗时统计

### 优化改进
- 本地批量导入与在线归档逻辑彻底隔离：
  - 批量导入只处理 `LocalModel_*`
  - 不与 `MW_*` 模型做合并或追加配置
- 导入后源文件自动归档：
  - 成功 / 重复文件移入 `_imported/`
  - 失败文件移入 `_failed/`
  - 后续扫描自动忽略这两个目录
  - 移动后会自动清理原路径空文件夹
- 配置页显示优化：
  - Cookie 更新时间格式化
  - Cookie 状态改为中文
  - 最近扫描结果改为中文显示

### Bug 修复
- 修复监控目录无更新时仍生成批量导入报告文件的问题
- 修复监控目录处理完成后缺少最近扫描状态展示的问题
- 修复批量导入通知标题未区分监控目录与手动目录来源的问题

## 兼容性与注意事项
- 本次新增 `local_batch_import` 配置块，老版本配置不会影响原有归档功能
- 推荐 Docker 用户增加 `./app/watch:/app/watch` 挂载
- 大规模导入优先使用 `watch` 目录 + 页面手动扫描，避免浏览器批量上传耗时过长
- 本地批量导入只对本地导入模型生效，不影响现有在线归档逻辑
