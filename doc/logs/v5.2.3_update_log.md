# v5.2.3 更新日志

> 更新日期: 2026-03-05

## 更新概述
本次为体验与可维护性修复版本，重点修复手动导入弹窗在暗黑模式下的显示问题，并为“归档更新”新增独立执行日志，便于排查跳过/失败/未定位明细。

## 详细变更

### 优化改进
- 修复手动导入弹窗暗黑主题样式，统一弹窗、输入框、富文本、按钮、草稿区的深色配色与对比度。

### 新增功能
- 新增归档更新独立日志 `rebuild_pages.log`：
  - 每次执行“归档更新”后自动追加一份执行报告；
  - 记录汇总信息（processed/updated/skipped/failed/fixed/unresolved）；
  - 记录 `跳过`、`失败`、`未定位实例` 的详细条目。
- `POST /api/archive/rebuild-pages` 返回中新增 `report_log` 字段，返回本次日志文件路径。

## 兼容性与注意事项
- 本次变更不影响既有归档数据结构；
- 新日志文件默认写入 `app/logs/rebuild_pages.log`，可随容器日志目录一起挂载持久化。

## 涉及文件
- `app/static/css/manual_import.css` - 补齐暗黑主题样式覆盖，修复对比度问题。
- `app/server.py` - 增加归档更新专用日志写入逻辑，并在接口返回追加 `report_log`。

## 验证记录
- 自测范围：暗黑模式下手动导入弹窗；执行一次“归档更新”并检查日志输出。
- 自测结果：通过，弹窗显示正常；`rebuild_pages.log` 已写入汇总与明细内容。
- 待验证项：不同模型量级下日志持续追加与容器挂载场景下的文件权限一致性。
