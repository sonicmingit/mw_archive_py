# v5.3 更新日志

> 更新日期: 2026-03-06

## 更新概述
本次为功能扩展版本，重点新增 Telegram 推送与机器人交互能力，并完成国内 / 国际平台 Cookie 分组支持。同时调整了 Docker 配置持久化方式，统一改为挂载 `app/config/` 目录，便于后续继续扩展通知与认证配置。

### v5.3 Docker 升级注意事项
如果你是从旧版本 Docker 升级到 `v5.3`，重点注意这次配置结构已经变化：
- 旧版常见挂载：
  - `./app/cookie.txt:/app/cookie.txt`
  - `./app/config.json:/app/config.json`
- `v5.3` 推荐改为直接挂载整个配置目录：
  - `./app/config:/app/config`
- 新目录下的关键文件：
  - `app/config/config.json`
  - `app/config/cookie.json`
  - `app/config/gallery_flags.json`
- 升级前建议先备份：
  - `app/data/`
  - `app/logs/`
  - `app/config/` 或旧的 `app/config.json`、`app/cookie.txt`

### Docker 升级步骤（推荐）
1. 停止并删除旧容器。
2. 备份宿主机上的 `app/data`、`app/logs`、`app/config`（如果没有 `app/config`，就备份旧的 `app/config.json` 和 `app/cookie.txt`）。
3. 将挂载方式改为：
   - `./app/data:/app/data`
   - `./app/logs:/app/logs`
   - `./app/config:/app/config`
4. 启动新版本容器。
5. 首次启动后检查 `app/config/` 目录：如果为空，程序会自动生成默认配置文件；如果存在旧配置，程序会自动迁移到新目录。
6. 打开 `/config` 页面确认：
   - Cookie 是否正常显示；
   - Telegram 配置是否保留；
   - 国内 / 国际平台 Cookie 是否分组正常。


## 详细变更

### 新增功能
- 新增 Telegram 推送与交互能力：
  - 支持归档成功推送；
  - 支持归档失败告警；
  - 支持 Cookie 失效 / 限流告警；
  - 支持机器人接收模型链接直接触发归档；
  - 支持 `/help`、`/cookies`、`/count`、`/search` 等基础命令。
- 新增统一通知分发层：
  - 业务代码统一调用通知分发器；
  - 当前接入 Telegram；
  - 为后续企业微信等通知渠道预留统一接入入口。
- 新增国内 / 国际平台 Cookie 支持：
  - 国内链接自动使用 `makerworld.com.cn` 对应 Cookie；
  - 国际链接自动使用 `makerworld.com` 对应 Cookie。

### 优化改进
- 配置目录统一调整为 `app/config/`：
  - `config.json`
  - `cookie.json`
  - `gallery_flags.json`
- Docker 部署推荐改为持久化整个 `app/config/` 目录，不再单独挂载 `cookie.txt`。
- 旧版 `app/config.json`、`app/cookie.txt`、`app/gallery_flags.json` 可在启动时自动迁移到新目录。
- Cookie 告警文案统一，失效和限流两类通知使用相同结构字段，便于后续扩展到其他通知渠道。

### Bug 修复
- 修复独立 Cookie 测试容易误判的问题：
  - 站点主页可能返回 `403`，并不能代表 Cookie 无法下载模型；
  - 已移除独立测试按钮，Cookie 状态改为只由真实归档 / 下载链路更新。
- 修复单 Cookie 模式下配置页显示 `Cookie 1` 的问题，现改为直接显示 `Cookie`。

## 兼容性与注意事项
- 本次是配置结构变更版本，Docker 用户升级时需重点检查挂载路径。
- 推荐新的 Docker 挂载方式：
  - `./app/data:/app/data`
  - `./app/logs:/app/logs`
  - `./app/config:/app/config`
- 不再推荐单独挂载 `app/cookie.txt`，因为 Cookie 已迁移到 `app/config/cookie.json`。
- 如果宿主机上的 `app/config/` 为空目录，服务启动后会自动生成默认文件，无需手动预建 `config.json` / `cookie.json`。
- 从旧版升级时，建议先备份：
  - `app/data/`
  - `app/logs/`
  - `app/config/` 或旧的 `app/config.json`、`app/cookie.txt`
