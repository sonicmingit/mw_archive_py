# Bug 跟踪说明与模板

本目录用于按“**一个 Bug 一个文档**”跟踪问题，便于长期维护与检索。

## 目录规则

- 文件命名：`BUG-YYYYMMDD-序号.md`
- 每个文件只记录一个 Bug
- 建议按时间和编号递增创建

## 状态说明

- `🟡 待确认`：问题已记录，但尚未完成复现确认
- `🛠️ 处理中`：已确认问题，正在分析或修复
- `🧪 待验证`：已提交修复，等待测试验证
- `✅ 已解决`：修复并验证通过
- `⚪ 已关闭`：确认无需处理（非 Bug/重复/无法复现）

## 记录模板（复制后新建文件）

```markdown
# [BUG-YYYYMMDD-序号] 标题（简述问题）

- 状态：🟡 待确认 / 🛠️ 处理中 / 🧪 待验证 / ✅ 已解决 / ⚪ 已关闭
- 严重级别：低 / 中 / 高 / 紧急
- 发现时间：YYYY-MM-DD HH:mm
- 解决时间：YYYY-MM-DD HH:mm（未解决填：-）
- 影响版本：vX.Y.Z（未知填：待确认）
- 解决版本：vX.Y.Z（未解决填：-）
- 记录人：姓名或ID
- 最后更新：YYYY-MM-DD

## 现象描述
- （用户可见问题，尽量描述清楚）

## 复现步骤
1. ...
2. ...
3. ...

## 期望结果
- ...

## 实际结果
- ...

## 初步分析 / 根因
- ...

## 解决思路
- ...

## 修复记录
- YYYY-MM-DD：做了什么改动（文件/PR/提交）

## 验证记录
- 验证人：
- 验证时间：
- 验证结果：通过 / 不通过
- 备注：

## 关联信息
- 关联链接：
- 关联文件：
```
